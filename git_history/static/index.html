    Commit History

    Shows the commits on your current branch, newest first.

    You can do the following rebase operations:

    • To select multiple commits, click the first one and shift-click the second. All commits in between will be selected. Press Escape to unselect.
    • Drag one or more commits up or down to reorder them. Press Escape to cancel.
    • Hover a commit to reveal the Fixup button (↵), which folds that commit into the one below it, discarding its commit message.
    • Double-click a commit message to edit it in place (Reword). Click outside the edit box or press Ctrl+Enter (Cmd+Enter on Mac) to confirm. Press Escape to cancel.
    • When two or more commits are selected, click the Squash button to combine them into a single commit where the commit message is the combined messages of all the commits.

    If a rebase produces a merge conflict, a dialog will appear letting you resolve the conflict in your editor and then continue, or abort and return to the original state.

    Tip: drag related commits together first, then use squash or fixup.

    Branch History

    Branch History is an undo stack for your local branch. Each entry is the tip of the branch, sorted by chronologically. The Branch History begins when the local branch was created.

    Each of the following actions moves the current branch and updates the Commit History pane:

    • Undo go one step backwards in Branch History.
    • Redo go one step forward in Branch History.
    • Double-clicking reset to given entry in the Branch History.

    The Branch History contains only unique modifications to the branch. If a branch is reset to an previous state, the new tip of the branch is not unique and will not be an entry in Branch History. Undo, Redo and Reset actions are non-unique modifications and are not added as new entries to the Branch History.

    Branch History shows the local branch changes — commits, rebases, resets — regardless of whether they were made inside or outside git-history.

    When you pull or fetch/rebase from the remote, the update to the local branch appears as a single entry in the Branch History, not as a series of individual remote commits.

    Tip: A failed rebase can be recovered by executing "git rebase --abort".

    Update submodules?

    Submodule pointers have changed. Click Update to run git submodule update --init.

    Refusing will leave the working tree dirty. git-history is disabled while the working tree is dirty.
